### 1. Tooling Fix: Registering `solidity-coverage` in Hardhat

**Problem**: Running `npx hardhat coverage` produced `Error HH303: Unrecognized task 'coverage'` even though `solidity-coverage` was listed in `package.json`.

**Root cause**: The plugin was installed as a devDependency but not *imported* in `hardhat.config.ts`. Hardhat only registers plugin CLI tasks when the module is explicitly imported.

**Fix** — added import to `hardhat.config.ts`:
```typescript
import "solidity-coverage";
```

After this fix, `npx hardhat coverage` executed cleanly and generated interactive Istanbul HTML reports at `coverage/index.html`.

### 4. ⚠️ Analysis of Uncovered Branches (Structurally Unreachable)

1. **`VaultManager.sol` (~10% residual branch gap)**:
   - Caused by OpenZeppelin `onlyOwner` and `whenNotPaused` modifier intermediate IR branch nodes generated by Solc. Sequential `require` statements produce branch pairs where secondary branches cannot be isolated without subverting modifier evaluation.
2. **`SavingCore.sol` (~4.88% residual branch gap)**:
   - The `require(deposit.status == DepositStatus.Active)` false-path in `withdrawAtMaturity` and `earlyWithdraw`.
   - **Reason**: When a deposit certificate is withdrawn or renewed, the ERC721 token is **burned**. Subsequent calls to `withdrawAtMaturity` or `earlyWithdraw` query `ownerOf(depositId)` first, which immediately reverts with `ERC721NonexistentToken` *before* the `deposit.status` guard can be evaluated. An existing NFT with a non-Active status is an EVM state impossibility in this architecture.

### 5. Smart Contract Deployment Script (`scripts/deploy.ts`)

Created a deterministic TypeScript deployment script (`scripts/deploy.ts`) for local node and live network deployments.

#### Execution Flow (`main()`)

1. **Deployer Setup**:
   - Queries `ethers.getSigners()` and logs deployer address and ETH balance.
2. **Deploy `MockUSDC.sol`**:
   - Constructor args: `name = "Mock USDC"`, `symbol = "mUSDC"`.
   - Result: 6-decimal test token deployed.
3. **Deploy `VaultManager.sol`**:
   - Constructor args: `usdcAddress`.
   - Sets deployer as owner and initial `feeReceiver`.
4. **Deploy `SavingCore.sol`**:
   - Constructor args: `usdcAddress`, `vaultAddress`.
   - Sets deployer as owner, initializes ERC721 name `"Deposit Certificate"`, symbol `"DEPOSIT"`.
5. **Wire Contract Permissions**:
   - Calls `vaultManager.setSavingCore(savingCoreAddress)`.
   - Grants `SavingCore` exclusive rights to `allocateInterest`, `cancelInterest`, and `payInterest`.
6. **Initialize Default Saving Plan #1 (Student ID `2231200021`)**:
   - Calls `savingCore.createPlan(tenorDays=90, aprBps=225, minDeposit=1000000, maxDeposit=1000000000000, earlyWithdrawPenaltyBps=400)`.
   - Configures default 90-day term plan (2.25% APR, 4.00% early withdraw penalty).
7. **Local Liquidity Setup (Hardhat / Localhost network only)**:
   - Mints `100,000 USDC` (`100_000_000_000` base units) to deployer.
   - Approves `VaultManager` and calls `vaultManager.fundVault(100_000 USDC)`.
   - Populates initial interest reserve pool for local testing and UI demonstration.

#### Package Command Shortcuts (`package.json`)
Added shortcut commands under `"scripts"`:
- `"deploy:localhost": "hardhat run scripts/deploy.ts --network localhost"`
- `"deploy:sepolia": "hardhat run scripts/deploy.ts --network sepolia"`
